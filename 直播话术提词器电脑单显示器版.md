# 一、现状问题复盘（基于现有截图+单屏直播场景）

当前产品左侧为竖向分类导航栏，话术层级为：**左侧点击分类→中间加载话术→鼠标点击选用**。该交互模式在直播高压快节奏下存在致命硬伤，完全不适用于临场应变：

1. **操作链路过长**：触发一句应急话术需要至少2~3次鼠标点击，耗时久，无法应对瞬时弹幕、突发黑粉、大额礼物。

2. **视线割裂严重**：主播视线需要在【主提词脚本、左侧分类栏、公屏弹幕】三处来回跳转，打断表演节奏、断语感。

3. **依赖精准鼠标操作**：直播过程中人手紧绷、操作仓促，鼠标极易误点、卡顿，容错率极低。

4. **无预判弹药储备**：当前必须先切换分类才能看到话术，主播无法提前预判、无法提前备好应急台词。

**核心结论：必须彻底废除左侧点击式分类栏，重构单屏视野层级，全部改为「主视野常驻+键盘零点击触发」机制。**

---

# 二、优化核心定位（单显示器专属）

## 2.1 设计理念

放弃传统侧边导航结构，采用**主脚本区+底部弹药带+右侧公屏AI区**三分固定布局。主播全程无需切换页面、无需点击侧边栏，高频话术永久常驻主视野，做到**眼睛不漂移、手不离开键盘、一秒响应弹幕**。

## 2.2 优化原则（硬性约束）

- **零鼠标依赖**：95%高频操作全部由键盘完成，鼠标仅用于后台编辑话术。

- **视野不割裂**：所有高频功能全部平铺在一屏，无二级弹窗、无隐藏侧边栏。

- **主次绝对分明**：中间主脚本为最高优先级，弹药带为辅，公屏AI区为补充。

- **无动画无延迟**：话术切换、高亮、刷新全部瞬时完成，杜绝卡顿动画干扰主播。

## 3.2 区域功能详细定义

### 3.2.1 中央主提词区（最高优先级）

保留原有全部脚本功能，不做删减改动，保障主播主线表演不中断：

- 大字号展示当前段子、口播脚本；
- 支持自动滚动、手动上下键翻页、滚轮调速；
- 空格键一键暂停/继续脚本，适配直播停顿节奏；
- 背景纯色、低干扰，杜绝花哨视觉元素。

### 3.2.2 右侧公屏+AI应答区（保留原版结构，轻度升级）

沿用现有右侧栏，不改动位置，降低改版学习成本：

- 实时滚动直播间弹幕，AI自动高亮：大额礼物、恶意黑粉、高频提问、带节奏弹幕；
- 针对高亮弹幕自动生成1条最优回复；
- **新增快捷操作**：AI推荐话术可一键存入话术库（待审核）；
- 支持临时收藏、快速加入个人话术库。

### 3.2.3 底部常驻话术弹药带（本次核心改版）

**彻底删除原左侧竖向分类导航栏，全部高频分类下移平铺常驻**，固定9大主播刚需分类，不可隐藏：感谢、回击、互动、带货、自定义、开播、闭播、抽奖、危机。

1. **卡片形态**：横向并排小卡片，自带颜色标识。
2. **展示规则**：每张卡片永久展示当前**待命单句话术**，无需点击、无需切换，余光可见。
3. **自动轮换**：无操作时，每类卡片自动静默刷新下一句，主播随时都是"满弹状态"。
4. **智能高亮**：触发对应直播行为自动闪烁边框提醒。

## 4.2 标准操作链路（以怼黑粉举例）

1. 右侧公屏出现恶意弹幕，AI自动高亮、回击卡片红色闪烁；
2. 手指轻按键盘【2】，瞬时切换合适回击金句；
3. 确认话术无误，可直接使用（话术仅供主播参考，不自动发送）；
4. 全程视线不离开主屏幕、手不离开键盘、无任何鼠标动作。

# 五、功能清单（已确认）

## 5.1 废除功能

- 废除左侧竖向分类导航栏（感谢、回击、互动、带货等全部一级分类按钮）；
- 废除点击分类刷新中间话术面板的旧逻辑；
- 废除话术通过 Enter 发送功能（话术仅供主播参考，不自动发送）；
- 废除脚本环节自动匹配弹药卡功能；
- 废除AI推荐一键填充弹药带功能；
- 废除话术不足提醒功能。

## 5.2 保留功能

- 中央主提词器全部脚本、自动滚动、手动翻页、调速、字号设置；
- 右侧公屏弹幕流、AI智能识别、基础回复推荐；
- 话术库后台增删改查、个人收藏、话术分组；
- AI推荐话术存入话术库（待审核）。

## 5.3 新增功能

1. **底部常驻九卡式话术弹药带**，彩色区分、永久常驻；
2. **全局无冲突快捷键系统**，支持自定义改键；
3. **AI弹幕联动卡片高亮闪烁提醒**；
4. **话术库自动补充弹药带**，用完自动从话术库补充；
5. **快捷键帮助弹窗**（?键触发）；
6. **多主题模式**，多套主题可切换；
7. **窗口弹幕抓取**（抖音直播伴侣）+ OCR兜底；
8. **主脚本按模板生成**，含时间节奏安排。

# 六、技术约束（研发必看）

1. **热键全局捕获**：软件最小化、直播软件置顶状态下，快捷键依旧生效，规避热键冲突，支持手动改键；

2. **零延迟刷新**：弹药带话术切换无动画、无渐变、无卡顿，点击瞬时刷新（≤50ms）；

3. **视觉层级管控**：主提词区透明度、字号优先级最高，弹药带透明度可调节，不抢主脚本视觉焦点；

4. **话术去重**：30分钟内相同话术不重复使用（可配置时间窗口）；

5. **AI响应策略**：缓存优先、话术库检索优先，库无可用话术再调用LLM生成；

6. **导入/导出**：支持text/md格式，LLM辅助解析分类和标签。

# 七、两种工作模式

## 7.1 直播模式

主界面布局，键盘驱动，零鼠标操作：

- 三区域可拖拽调整位置和大小；
- 底部9个话术分类弹药带常驻；
- 右侧公屏实时弹幕+AI推荐；
- 中央主脚本自动滚动+空格暂停+上下键翻页。

## 7.2 维护模式

独立完整页面，数据管理后台：

- 话术管理：增删改查、待审核话术、批量操作、AI改写、AI批量新增；
- 模板编辑：拖拽调整顺序、删除、新增、段落时长配置；
- 槽位配置：9个槽位与分类的映射关系；
- AI配置：多Provider支持（baseURL + apiKey + model + 优先级）；
- 弹幕抓取配置：窗口选择、抓取频率（2s/5s）、OCR设置；
- 主题配置：多套主题切换；
- 系统设置：热键配置、透明度、去重时间窗口等；
- 导入导出：text/md格式 + LLM转换。
